📋 Changelog — Perubahan Schema

🆕 v1.1 — Penambahan Tabel Progress Belajar

  • enrollments — Tabel baru untuk mencatat enrollment user ke course setelah pembayaran berhasil. Memiliki relasi ke users, courses, dan orders. Constraint UNIQUE(user_id, course_id) mencegah duplikasi enrollment.
  • lecture_completions — Tabel baru untuk tracking progress belajar per lecture. Memiliki relasi ke users dan course_lectures. Constraint UNIQUE(user_id, lecture_id) memastikan satu record per lecture per user.
  • Total FK bertambah dari 24 menjadi 28 foreign keys.
  • Total tabel bertambah dari 17 menjadi 19 tabel.

📌 v1.0 — Schema Awal

  • users — Ditambahkan kolom: role, photo, phone, address, bio, website (extend dari Laravel default).
  • courses — Tabel utama dengan relasi ke categories, sub_categories, dan users (instructor). Status enum: draft/pending_review/active/inactive.
  • payments — Integrasi Midtrans dengan kolom midtrans_order_id (UNIQUE), midtrans_transaction_id, payment_type, dan midtrans_response (JSON).
  • orders — Denormalisasi instructor_id untuk performa query laporan penjualan tanpa JOIN ke courses.
  • coupons — Coupon bisa global (course_id NULL) atau spesifik per course (SET NULL on delete).
  • reviews — Rating tinyint 1-5 dengan UNIQUE(user_id, course_id) — satu review per user per course.
  • CMS tables: sliders, info_boxes, partners, site_infos — tidak memiliki FK (standalone).
